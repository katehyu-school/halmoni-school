# Basics 온램프 설계 — Beginner Onramp

> 작성일: 2026-05-21 (Opus 세션)
> 목적: 한국어를 **전혀 모르는 사람**을 자신감 있게 Beginners(ep1) 코스로 진입시키는 step-by-step 가이드
> 핵심 철학: 작은 성공의 연속 → 자신감 → ep1 진입

---

## 🎯 해결하려는 문제

**현재 상황:**
- Beginners ep1은 "안녕? 나는 정민이야"로 시작 — 이미 한글을 읽을 수 있다고 가정
- 하지만 K-pop 보고 온 진짜 초보는 **한글도 못 읽음**
- Basics에 한글/숫자 등 자료는 있지만, **어디서 시작해 어떻게 ep1까지 갈지 길이 없음**

**온램프의 역할:**
- "한글 못 읽음" → "ep1 시작 가능" 까지의 **명확한 단계별 경로**
- 매 단계 작은 성취 → 이탈 방지 (선생님 표현: "도망가지 않게")

---

## 🧭 온램프 5단계 (Onramp Stages)

```
START (한국어 0)
   │
   ▼
[1] 한글 읽기      ← 글자를 소리로 (가장 큰 장벽)
   │
   ▼
[2] 발음 감각      ← 받침·연음 맛보기 (가볍게)
   │
   ▼
[3] 생존 표현      ← 안녕하세요, 감사합니다 (즉시 써먹기)
   │
   ▼
[4] 문장 구조 맛보기 ← SOV·조사 1분 소개 (ep5 예고편)
   │
   ▼
[5] 숫자 맛보기     ← 1~10 + 기본 (ep3/4 대비)
   │
   ▼
🎉 READY → Beginners ep1 진입!
```

### 단계별 상세

#### [1] 한글 읽기 (Reading Hangul) — 가장 중요
- **목표**: 자음 14 + 모음 10 + 받침 → 음절 블록 읽기
- **기존 자산**: Basics > hangul 토픽 (CONSONANTS, VOWELS, combineSyl 음절 조합기)
- **온램프 추가**:
  - "오늘은 딱 5글자만!" 식의 **마이크로 목표** (한 번에 다 X)
  - 읽기 미니 게임: 글자 보고 소리 맞히기
  - 진도 표시 (자음 7/14 익힘 등)
- **졸업 기준**: 자기 이름을 한글로 읽을 수 있다

#### [2] 발음 감각 (Pronunciation Feel) — 가볍게
- **목표**: "쓰는 것 ≠ 읽는 것" 인지 (연음/받침 7대표음 맛보기)
- **기존 자산**: nhs pronunciation 섹션, Basics speaking 토픽
- **온램프 버전**: 깊이 X, "한국어는 소리가 이어진다" 정도만 (예: 먹어요 [머거요])
- **졸업 기준**: 연음 개념을 안다 (완벽한 발음 X)

#### [3] 생존 표현 (Survival Phrases) — 즉시 보상
- **목표**: 당장 써먹을 수 있는 10개 표현
- **내용**: 안녕하세요 / 감사합니다 / 죄송합니다 / 네·아니요 / 잘 먹겠습니다 / 안녕히 계세요...
- **온램프 효과**: "벌써 한국어 한다!" 즉각적 성취감 (TTS 듣고 따라하기 + 말해보기 기능)
- **졸업 기준**: 기본 인사 5개를 듣고 안다

#### [4] 문장 구조 맛보기 (Sentence Structure Teaser) — ep5 예고
- **목표**: 한국어가 영어와 다르다는 큰 그림 (SOV, 조사) — **1분 분량**
- **주의**: 깊이 X! ep5에서 제대로 다룸. 여기선 "맛보기"만
- **내용**: "영어는 I eat rice, 한국어는 나는 밥을 먹어요 (먹다가 끝에!)"
- **졸업 기준**: 동사가 끝에 온다는 걸 안다

#### [5] 숫자 맛보기 (Numbers Teaser) — ep3/4 대비
- **목표**: 1~10 (한자어 + 고유어 존재 인지)
- **기존 자산**: Basics > numbers (BASICS_NUMBERS_DESIGN.md 큰 숫자는 reference로)
- **온램프 버전**: 1~10만, 큰 숫자는 "나중에 Basics에서"
- **졸업 기준**: 1~10 셀 수 있다

## 🎨 UI/UX 설계

### 온램프 진입 화면
- Basics 탭 최상단에 **"🚀 처음이세요? 여기서 시작!"** 큰 버튼
- 클릭 → 온램프 5단계 체크리스트 화면

### 진도 시각화 (핵심!)
```
🚀 Hangeul Quest 온램프

[✅] 1. 한글 읽기        ████████░░ 80%
[🔄] 2. 발음 감각        ███░░░░░░░ 30%
[  ] 3. 생존 표현        ░░░░░░░░░░  0%
[  ] 4. 문장 구조 맛보기  ░░░░░░░░░░  0%
[  ] 5. 숫자 맛보기      ░░░░░░░░░░  0%

      [ ep1 시작하기 → ]  (3단계 이상 완료 시 활성화)
```

### 설계 원칙
1. **마이크로 목표**: "오늘 5글자" 식 — 부담 최소화
2. **즉각 보상**: 각 단계 완료 시 🎉 축하 + 진도 채워짐
3. **건너뛰기 허용**: 이미 한글 아는 사람은 skip 가능 (게이트 아님)
4. **ep1 게이트는 느슨하게**: 3단계(생존표현)까지만 해도 ep1 진입 허용 (강제 X)

## 🔗 기존 구조와의 연결

| 온램프 단계 | 기존 Basics 토픽 | 재활용 |
|------------|-----------------|--------|
| 1 한글 | hangul (자음/모음/음절조합) | ✅ 그대로 + 마이크로 목표 래핑 |
| 2 발음 | speaking, pronunciation | ✅ 가벼운 버전 |
| 3 생존표현 | greetings | ✅ + 말해보기(Claude API) |
| 4 문장구조 | structure | ✅ 1분 요약 버전 |
| 5 숫자 | numbers | ✅ 1~10만 (큰 숫자는 reference) |

→ **온램프는 새 콘텐츠가 아니라 기존 Basics를 "순서 + 진도 + 동기부여"로 재포장**하는 것!

## 🏗️ 구현 단계 (제안)

### Phase 1: 온램프 데이터 구조
```javascript
const ONRAMP = [
  { id:'hangul', title:'한글 읽기', title_en:'Read Hangul', icon:'🔤',
    goal:'자기 이름을 한글로 읽기', topic:'hangul', minGames:5 },
  { id:'pron', title:'발음 감각', ... topic:'speaking' },
  { id:'survival', title:'생존 표현', ... topic:'greetings' },
  { id:'structure', title:'문장 구조 맛보기', ... topic:'structure' },
  { id:'numbers', title:'숫자 맛보기', ... topic:'numbers' },
];
```

### Phase 2: 진도 추적
- localStorage에 단계별 완료율 저장 (브라우저 단위)
- 단순 체크박스 방식부터 시작 (게임 점수 추적은 나중에)

### Phase 3: 진입 화면 + 진도 바 UI
- Basics 상단 "처음이세요?" 버튼 + 온램프 화면

### Phase 4: ep1 게이트 연동
- 온램프 3단계 이상 완료 → ep1 "추천" 배지

## 💡 핵심 통찰

> **온램프 = 기존 Basics에 "길"을 깔아주는 것.**
> 콘텐츠를 새로 만드는 게 아니라, 초보가 헤매지 않게 **순서 + 진도 + 작은 성취**를 더하는 설계.
> "도망가지 않게" 하려면 → 부담은 줄이고, 성취감은 자주.
